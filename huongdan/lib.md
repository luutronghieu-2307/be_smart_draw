Chào ông, đây là danh sách "tận răng" toàn bộ các thư viện (Tech Stack n-1) ông cần để dựng cái hệ thống này. Tui chia theo từng cụm để ông dễ quản lý và quăng vào file cấu hình (pom.xml, build.gradle, pubspec.yaml).1. Backend: Java Spring Boot (File pom.xml)Đây là những thứ ông cần khai báo trong dự án Maven để chạy Server API.Tên thư việnPhiên bản (n-1)Chức năng chínhSpring Boot Starter Web4.0.4Dựng REST API, hỗ trợ Virtual Threads xử lý RTSP.SpringDoc OpenAPI (Swagger)2.3.0Tự động tạo giao diện UI để ông test API tại chỗ.ONNX Runtime (GPU)1.23.4"Não" AI. Dùng bản GPU để sau này có máy mới không cần đổi code.OpenCV Java Bindings4.12.0Xử lý ma trận ảnh, bắt luồng RTSP qua GStreamer.Eclipse Paho MQTT1.2.5Kết nối và gửi lệnh xuống cho ESP32.Lombok1.18.34Giúp code Java sạch hơn (giảm bớt Getter/Setter).2. Hệ điều hành & Docker (Phần "Móng" trên Pop!_OS)Ông không cài trực tiếp vào máy mà nên khai báo trong Dockerfile hoặc cài trên Pop!_OS để hỗ trợ Docker.NVIDIA Container Toolkit: Để Docker "thông" với GPU (khi ông sửa được máy hoặc mua máy mới).GStreamer Core (v1.26.x): Cần các plugin: gst-plugins-base, gst-plugins-good, gst-plugins-bad, gst-libav.Base Docker Image: nvidia/cuda:12.4.1-runtime-ubuntu22.04 (Bản ổn định nhất để chạy Java + AI).MQTT Broker: eclipse-mosquitto:2.0.20 (Chạy bằng Docker Compose).3. Frontend: Mobile App (File pubspec.yaml của Flutter)Dành cho con Redmi Note 13 của ông để điều khiển ngăn kéo.LibraryPhiên bảnChức năngFlutter SDK3.41.xBản stable n-1 cực kỳ mượt.Dio5.7.0Thư viện gọi API mạnh mẽ hơn http thuần.MQTT Client10.7.xNếu ông muốn App nhận thông báo trực tiếp từ ESP32.Video Player2.9.xĐể xem luồng RTSP trực tiếp trên điện thoại.4. IoT: ESP32 (Phần cứng "Ngăn kéo")Tùy vào việc ông code bằng ESP-IDF (Expert) hay Arduino IDE (Nhanh).Nếu dùng ESP-IDF (Khuyên dùng): Dùng Component mqtt có sẵn trong SDK của Espressif.Nếu dùng Arduino:PubSubClient (v2.8): Để kết nối MQTT.ArduinoJson (v7.x): Để đọc lệnh đóng/mở cửa gửi từ Java Server.5. Công cụ bổ trợ (Tooling)Postman/Insomnia: Để test API (nếu không dùng Swagger).MQTT Explorer: Cực kỳ quan trọng để ông "soi" xem lệnh từ Java có thực sự bay tới ESP32 hay không.VLC Player: Để check luồng RTSP từ Camera có sống hay không trước khi nạp vào Java.Tổng kết "Combo" ưu tiênVì ông đang làm một mình, tui khuyên ông tập trung vào Backend Java trước với 3 món chính: Spring Boot 4.0.4 + ONNX 1.23.4 + OpenCV 4.12.0. Chỉ cần 3 món này là ông đã xong 80% phần xử lý AI rồi.Ông muốn tui bắt đầu viết file pom.xml mẫu chứa tất cả các thư viện Java này để ông copy vào dự án luôn không?
